# Fork
Uproszczony schemat rozwijania oprogramowania OpenSource
1. Fork - najprościej przez kliknięcie przycisku Fork w githubie - tworzy własna kopię repozytorium zdalnego
2. Klonowanie repozytorium zdalnego na maszynę lokalną
3. Stworzenie brancha roboczego
4. Wprowadzenie zmian w kodzie
5. Połączenie zmian z kodem z maina i sprawdzenie go na wypadek wystąpienia konfliktów
6. Wypchnięcie zmienionego (działającego) kodu na swoje zdalne repozytorium
7. Stworzenie pull requesta
8. Po zaakceptowaniu pull requesta - usunięcie zbędnego forka.
